## 🧠 Dev Notes & Hypotheses (Bugs, ideas, context)
- **Engine map:** `Simple_Search` ctor (`simple-search/v1.1/simple_search.php`)
  = convert_nonascii → dalnorm.normalize → doVariant recursion over
  `transitionTable_default` (broad) pruned by `ngramValidate` (LIKE 'pref%') →
  generate_alternate_endings + grammar_variants → kept iff normalizes to a
  hwnorm1c headword → order_by_wf. JSON built in `getword_list_1.0_main.php`.
- **Overgeneration root cause:** equivalence classes (nasals 6-way, sibilants,
  r-cluster 9-way, b/v) applied unconditionally + bidirectionally at every
  position; validation is existence-only (no distance/score); endings+grammar
  multiply again. Live MW counts (input=default, 2026-06-11): agni=1, deva=5,
  kara=11, sana=18, **manas=20**, xqzqxq=0. These are real, pulled from the
  live `getword_list_1.0.php`.
- **KEY CORRECTION (verified live):** overgeneration is **`default`-mode ONLY**.
  `restrict_to_user_word` (2021) returns just the exact word for any non-default
  input mode when that word exists — `input=iast` manas/sana/kara/saMskfta each
  return **1** (vs 20/18/11/— under default). So Fix A (precise table) is a
  *minor* lever; the headline fix is the default-mode score + hard-drop (Fix B /
  milestone M2). Both docs were corrected to say this after an initial overstatement.
- **Input coverage:** `convert_nonascii` auto-detects only Devanagari + Cyrillic.
  `utilities/transcoder/wx_slp1.xml` EXISTS but is unwired. Velthuis + all
  non-Devanagari Brahmic tables are absent. No NFC pass. Capital letters: default
  path `mb_strtolower`s everything (good for romanization, lossy for HK/SLP1/
  upper-ITRANS typed in default mode; combining marks on capitals need NFC).
- **Decisions locked (2026-06-11 feedback round):** trust precise input (no
  nasal/sibilant fuzz for slp1/deva/iast); result policy = score + HARD-DROP
  (not collapse, not reorder-only); add all four input tracks (loose-ASCII+case,
  Brahmic, WX+Velthuis, ISO-15919/NFC/case). Roadmap = `simple-search/roadmap_v1.2.md`.
- ASCII schemes (WX/Velthuis/HK) can't be auto-detected (mutually ambiguous) →
  explicit `<select>` options only; only NON-ASCII scripts go in the detect loop.

  harness, all four streams. Frame = simple-search is the discovery/retrieval
