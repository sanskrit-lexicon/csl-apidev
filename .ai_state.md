## 🧠 Dev Notes & Hypotheses (Bugs, ideas, context)
- Resolver plumbing already exists — this is only a routing layer:
  `dal->get2(lnum,lnum)` (`dal.php:158`) does id→record; `listhier`/`getword`
  already honor `lnum` (wins over `key`); `/simple/` already does positional path
  parsing (`simple-search/v1.1/parse_uri.php` + `list-0.2s_rw.php`). `cleanurl.php`
  = that machinery generalized from the fixed `simple` prefix to a dict-keyed one.
- Disambiguation is purely lexical: first post-dict segment matching `^\d+(\.\d+)?$`
  → id route; else headword; trailing `^\d+$` → homonym index. slp1 headwords are
  never all-numeric, so it's unambiguous.
- **Caveat (documented, §9):** `lnum` is line-derived and can shift on dictionary
  regeneration → id links are convenience aliases; headword form is the durable one.
- Top-level `/MW/...` chosen (per #249) despite collision risk; guarded by a
  dict-code whitelist in the rewrite so non-dict root paths pass through untouched.
- **README regeneration hazard fixed:** README is auto-generated by the Cologne
  Tooling Runbook. The `## Documentation` block is now wrapped in
  `<!-- BEGIN MANUAL: documentation -->` / `<!-- END MANUAL: documentation -->`
  guard markers, and BOTH runbook skill files (`~/.claude/commands/
  cologne-tooling-runbook.md` and `cologne-issue-runbook.md`, Phase 9 item 2) were
  hardened to preserve marked blocks verbatim AND to match `doc/` (singular), not
